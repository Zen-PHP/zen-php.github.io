nginx根目录下php-403错误

发表于 2016-08-11

之前好不容易通过Vagrant+VirtualBox搭建好了PHP运行环境，但是在宿主机访问虚拟机服务器根目录html下的php项目时，一直报403 Forbidden错误，但是直接访问php单个文件却是毫无问题，太奇怪了

问题描述

  • 肯定不是程序错误或者逻辑错误
  • 403确定是文件或者目录权限问题
  • 通过修改文件或者目录权限为777测试，发现不是纯粹的文件权限问题

解决办法

  • 编辑文件/etc/nginx/sites-available/default
  • 修改server方法里的目录权限设置参数
  • 添加index.php，修改后正确的参数如下：
    index index.php index.html index.htm;
  • 然后测试检查配置文件
    sudo nginx -t
  • 然后重新加载配置文件
    sudo service nginx reload
  • 最后重启nginx
    sudo service nginx restart
