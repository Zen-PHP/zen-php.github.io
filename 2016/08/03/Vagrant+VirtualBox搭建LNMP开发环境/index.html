Vagrant+VirtualBox搭建LNMP开发环境

发表于 2016-08-03

标签（空格分隔）： 工具

经历过windows系统下WAMP等集成开发环境和VirtualBox虚拟机运行Linux系统开发之后，发现Vagrant+virtualbox搭建PHP运行环境是比较好的

资源下载

（一）下载VirtualBox

  • 官方下载：VirtualBox

    （二）下载Vagrant

  • 官方下载：Vagrant

    （三）下载系统镜像

  • 官方下载：VagrantBox
  • 推荐版本：Official Ubuntu 12.04 daily Cloud Image amd64 (VirtualBox 4.1.12)

安装配置

（一）安装

1、安装VirtualBox+Vagrant

PS:安装vagrant之后需要重启电脑完成环境变量设置等配置

2、新建工作目录，如：

`D:VagrantWorkcentos-6.6-x86_64`

3、打开CMD命令行，进入新建工作目录

` D:`
`cd D:VagrantWorkcentos-6.6-x86_64`

4、初始化命令：

`vagrant init centos6.6`

5、手动加载BOX

  • 把下载的centos-6.6-x86_64.box复制到本目录D:VagrantWorkcentos-6.6-x86_64下，执行:
    vagrant box add centos6.6 centos-6.6-x86_64.box

6、BOX列表（检查是否导入成功）：

`vagrant box list`

（二）配置

PS:安装系统成功之后，会自动生成一个vagrantfile文件，是系统配置文件

1、端口映射：

`config.vm.network :forwarded_port, guest: 80, host: 8080`
上面这句代码前面的#号去掉。它表示映射本机的8080端口到虚拟机的80端口

2、目录映射：

config.vm.synced_folder "D:/www", "/var/www/html"
表示把本机的data目录共享到虚拟机里的/var/www目录

基本操作

  1. 启动
    vagrant up
  2. 关机
    vagrant halt
  3. 暂停
    vagrant suspend
  4. 恢复
    vagrant resume
  5. 移除
    vagrant box remove centos6.6
  6. 删除
    vagrant destroy
  7. 导出
    vagrant package
  8. 连接
  • DOS下vagrant ssh直接连接
  • 第三方终端模拟器连接（windows下支持）

安装Nginx+MySQL+PHP

apt方式安装Nginx+mysql+php

安装Nginx

sudo apt-get install nginx
  1. 所有的配置文件都在/etc/init.d/nginx这个目录下
  2. 修改默认配置文件
    sudo vim /etc/nginx/sites-available/default

    3.测试配置文件
    sudo nginx -t
    4.重载配置文件
    sudo service nginx reload

    安装MySQL

    1.安装MySQL服务端
    sudo apt-get install mysql-server
    然后你会看见输入密码的界面，你只需接下来输入管理员密码就行
    2.安装MySQL客户端
    sudo apt-get install mysql-client
    3.安装php5-fpm
    sudo apt-get install php5-mysql
    4.修改 mysql 的配置文件
    sudo vim /etc/mysql/my.cnf

    将 bind-address = 127.0.0.1 注释掉，就可以远程连接数据库了

5.查看MySQL状态
sudo netstat -tap | grep mysql
6.启动MySQL
sudo service mysql start
7.Mysql服务器开启远程连接

  • 开启对应账号的远程访问权限
    mysql -uroot -p
    use mysql;
    grant all privileges on . to ‘帐号’@’%’ identified by ‘密码’;
    flush privileges;
    如果在执行update的时候报错 “ERROR 1062 (23000): Duplicate entry ‘%-root’ for key ‘PRIMARY’”,解决方法如下：
    delete from user where user = ‘root’ and host = ‘*’;
  • 开启服务允许远程连接，编辑文件
    sudo vim /etc/mysql/my.cnf
  • 重启mysql服务
  1. SQLyog连接数据库
    主机地址：vagrantfile里设置的IP
    端口：3306

    配置虚拟主机域名

  • 编辑C:\Windows\System32\drivers\etc\hosts文件
  • 绑定Vagrant已有IP至虚拟主机域名

    安装phpmyadmin

    1.安装
    sudo apt-get install phpmyadmin
    phpmyadmin默认安装在/usr/share/下，所以需要建立软链接
    2.建立软链接
    sudo ln -s /usr/share/phpmyadmin phpmyadmin

问题BUG

报错 ： C:/HashiCorp/Vagrant/embedded/gems/gems/vagrant-1.8.4/lib/vagrant/util/platform.rb:85:in block in windows_hyperv_admin?': undefined methodinclude?’ for # (NoMethodError)
from C:/HashiCorp/Vagrant/embedded/gems/gems/vagrant-1.8.4/lib/vagrant/util/platform.rb:89:in call' from C:/HashiCorp/Vagrant/embedded/gems/gems/vagrant-1.8.4/lib/vagrant/util/platform.rb:89:inwindows_hyperv_admin?’
from C:/HashiCorp/Vagrant/embedded/gems/gems/vagrant-1.8.4/plugins/providers/hyperv/provider.rb:20:in usable?' from C:/HashiCorp/Vagrant/embedded/gems/gems/vagrant-1.8.4/lib/vagrant/environment.rb:381:inblock in default_provider’
from C:/HashiCorp/Vagrant/embedded/gems/gems/vagrant-1.8.4/lib/vagrant/environment.rb:379:in `each’
解决 ：造成此问题的原因可能是由于VirtualBox版本问题，最新的5.1可能会有此问题，建议安装5.0版 解决方案

2.卸载MySQL
sudo apt-get autoremove --purge mysql-server-5.0
sudo apt-get remove mysql-server
sudo apt-get autoremove mysql-server
sudo apt-get remove mysql-common
dpkg -l |grep ^rc|awk '{print $2}' |sudo xargs dpkg -P

3.在修改my.cnf文件权限的时候，如果将权限修改为777，那么就意味着权限全局可写，任何一个用户都可以写，mysql担心这种文件被其他用户恶意修改，所以忽略掉这个配置文件这就会导致用户在关闭或者重启mysql的时候无法实施，进而报错.
解决：sudo chmod -R 644 my.cnf
